## 1. 개요

### 1.1 배경 및 동기

임상 현장에서는 ECG, EEG, PPG, ABP, EMG, 호흡 등 다양한 생체신호가 환자 모니터링에 활용되고 있다. 기존 연구들은 대부분 **단일 모달리티**(ECG만, EEG만)에 특화된 모델을 개발해왔으며, 여러 종류의 생체신호를 통합적으로 이해하는 범용 모델은 아직 충분히 탐구되지 않은 영역이다.

본 프로젝트는 **다종의 생체신호를 단일 모델로 학습하는 Foundation Model**의 개발을 목표로 한다. ECG, EEG, EMG, 호흡 등 임상에서 활용되는 연속 파형(continuous waveform) 신호를 신호 종류에 구애받지 않고 통합 처리하며, 자기지도학습(Self-Supervised Learning)을 통해 대규모 비라벨 데이터에서 범용적인 생체신호 표현(representation)을 학습하고, 이를 다양한 하류 임상 과제(downstream task)에 전이(transfer)하는 것을 지향한다.

### 1.2 설계 철학

본 모델의 핵심 설계 원칙은 다음과 같다:

1. **Any-Signal, Any-Channel**: 신호 종류와 채널 수에 구애받지 않는 범용 입력 처리
2. **Any-Sampling Rate**: 생체신호마다 다른 샘플링 레이트(예: ECG 500Hz, EEG 256Hz, Resp 25Hz)를 별도의 리샘플링 없이 수용하는 구조
3. **효율적 가변 처리**: 가변 길이, 가변 채널 수를 패딩 없이 효율적으로 처리 (Packed Batch)
4. **Curriculum Learning**: 단순한 과제에서 복잡한 과제로 점진적 학습 (Channel-Independent → Any-Variate)
5. **Cross-Modal Understanding**: 서로 다른 생체신호 간의 상관관계 학습 (예: ECG ↔ PPG)
6. **Few-Shot Ready**: LLM처럼 소수의 라벨 데이터만으로 새로운 임상 과제에 즉시 적용 가능한 범용 표현 학습

### 1.3 기존 연구와의 비교

| 모델 | 신호 | 입력 길이 | Multi-Modal | Downstream Task |
|------|------|---------|-------------|-----------------|
| BIOT (NeurIPS 2023) | EEG/ECG/EMG | 10초 | O (학습은 별도) | Abnormal detection, Event type classification, Seizure detection/classification, Arrhythmia detection, HAR |
| LaBraM (ICLR 2024) | EEG | 가변 | X | Abnormal detection, Event type classification, Emotion recognition, Gait prediction |
| FoME (2024) | EEG | 90초 | X | Seizure detection/classification, Emotion recognition, Sleep staging, Forecasting, Imputation |

본 모델의 주요 차별점은 (1) 다종 생체신호의 **통합 사전학습** (신호 종류 수에 제한 없는 확장 가능 구조), (2) **Cross-modal reconstruction + contrastive learning**을 통한 신호 간 관계 학습, (3) **2-Phase Curriculum**을 통한 점진적 학습 전략으로 볼 수 있다.

## 2. 모델 아키텍처

### 2.1 전체 파이프라인

```
Raw Signal (variable length, variable channels)
    │
    ▼
┌─────────────────────────────────────────────┐
│  Packed Batch (bin-packing, no padding)      │
└─────────────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────────────┐
│  Scaler (PackedAbsMeanScaler)               │
│  - Per-variate 정규화                        │
│  - Outlier-robust (평균 절대값 기반)           │
└─────────────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────────────┐
│  Patch Embedding                             │
│  - 연속 신호 → 고정 크기 패치 토큰화           │
│  - Linear Projection 또는 CNN Stem           │
└─────────────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────────────┐
│  Dual Additive Embedding                     │
│  - Signal Type Embedding (N종, 확장 가능)     │
│  - Spatial ID Embedding (M개 위치, 확장 가능)  │
└─────────────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────────────┐
│  Transformer Encoder                         │
│  - Grouped Query Attention (GQA)             │
│  - Rotary Position Embedding (RoPE)          │
│  - Gated Linear Unit FFN (GLU)               │
│  - RMSNorm                                   │
│  - Binary Attention Bias (variate 구분)       │
└─────────────────────────────────────────────┘
    │
    ├── task="masked"    → Reconstruction Head + Cross-Modal Head + Contrastive Projection
    └── task="next_pred" → Next-Patch Prediction Head
```

### 2.2 핵심 구성요소

#### 2.2.1 Packed Batch & Bin-Packing Collate

생체신호 데이터는 신호 종류마다 채널 수와 샘플링 레이트가 다르다. 기존 방식처럼 최대 길이에 맞춰 패딩하면 GPU 메모리가 낭비될 수 있다. 본 모델은 **First-Fit Decreasing (FFD) bin-packing** 알고리즘으로 서로 다른 길이의 variate를 하나의 텐서에 효율적으로 패킹하는 방식을 채택하였다.

```
Sample A: ECG 2ch, 7680 samples  →  variate 2개
Sample B: EEG 3ch, 7680 samples  →  variate 3개
Sample C: Resp 1ch, 7680 samples →  variate 1개

┌─────────────────────────────────────┐
│ Packed Tensor (6 variates, 7680)    │
│ [A_ECG1 | A_ECG2 | B_EEG1 | ...]   │
│ + sample_id, variate_id 메타데이터    │
└─────────────────────────────────────┘
```

이를 통해 **패딩 없이** 가변 채널/길이를 처리할 수 있으며, GPU 활용률 향상을 기대할 수 있다.

#### 2.2.2 Dual Additive Embedding

생체신호의 **종류**(signal_type)와 **측정 위치**(spatial_id)를 모두 인코딩하기 위해 이중 임베딩 체계를 설계하였다.

| 구분 | 역할 | 예시 |
|------|------|------|
| **Signal Type** (대분류) | 신호의 물리적 특성 구분 | ECG, ABP, EEG, PPG, EMG, Resp 등 (확장 가능) |
| **Spatial ID** (소분류) | 측정 위치/전극 구분 | EEG Fp1, EEG C3, ECG Lead II 등 (확장 가능) |

두 임베딩은 **가산(additive)** 방식으로 결합되어 패치 임베딩에 더해진다:

```
patch_embedding += signal_type_embed[type_id] + spatial_id_embed[global_spatial_id]
```

이 설계를 통해 모델이 "이 패치가 어떤 종류의 신호이고, 어디서 측정되었는가"를 인지하면서도 **모든 신호를 하나의 시퀀스로 통합 처리**할 수 있을 것으로 기대한다.

#### 2.2.3 Transformer Encoder

최신 언어 모델에서 검증된 구성요소들을 채택하였다:

- **Grouped Query Attention (GQA)**: Multi-Head Attention 대비 메모리 효율적이면서 성능을 유지하는 것으로 알려져 있다 (Llama 2에서 검증)
- **Rotary Position Embedding (RoPE)**: 상대적 위치 정보를 회전 행렬로 인코딩하여 가변 길이에 자연스럽게 대응할 수 있다
- **Gated Linear Unit FFN (GLU)**: 게이팅 메커니즘으로 정보 흐름을 제어하여 학습 안정성 향상에 기여할 수 있다
- **Mixture of Experts (MoE)**: GLU FFN을 expert로 사용하는 Sparse MoE 구조를 선택적으로 적용할 수 있다. Centroid 기반 라우팅으로 각 토큰과 expert centroid 간 거리(cdist)를 계산하고, top-k expert만 활성화하여 출력을 가중합산한다. 파라미터 수를 늘리면서도 토큰당 연산량은 일정하게 유지할 수 있어, 다종 생체신호처럼 이질적인 입력 분포를 처리하는 데 유리할 것으로 기대된다. 각 expert가 특정 신호 종류나 주파수 대역에 자연스럽게 특화될 가능성이 있다 (Salesforce uni2ts에서 차용)
- **RMSNorm**: LayerNorm 대비 계산 효율적이면서 동등한 성능을 보이는 것으로 보고되고 있다
- **Binary Attention Bias**: Packed Batch에서는 여러 variate의 패치가 하나의 시퀀스에 혼합되므로, query와 key의 variate_id가 같은지(intra-variate) 다른지(inter-variate)에 따라 학습 가능한 bias를 더해준다. Head마다 독립적인 bias를 학습하여, 일부 head는 시간적 패턴(intra-variate)에, 다른 head는 모달리티 간 관계(inter-variate)에 집중하는 역할 분담이 가능하다 (Salesforce uni2ts에서 차용)

#### 2.2.4 Modality-Specific CNN Stem (선택적)

선형 프로젝션(Linear) 대신 **신호 종류별 전용 1D-CNN**으로 저수준 특징을 추출하는 옵션을 제공한다.

```
Raw Patch (P samples) → Conv1d(1→64, k=3) → GELU → Conv1d(64→64, k=3) → GELU → Conv1d(64→d_model, k=1) → Pool → (d_model,)
```

- 신호 종류별 독립적인 필터를 학습하여 ECG의 QRS complex, EEG의 spindle 등 모달리티 고유 패턴을 포착할 수 있을 것으로 기대한다
- 공유 CNN과 분리 CNN 모두 지원하며, 효과는 실험적으로 비교할 예정이다

## 3. 학습 전략

### 3.1 2-Phase Curriculum Learning

생체신호 학습을 두 단계로 나누어 점진적으로 복잡도를 높이는 전략을 채택하였다.

```
═══════════════════════════════════════════════════════════════════
  Phase 1: Channel-Independent (CI) Pre-training
═══════════════════════════════════════════════════════════════════

  입력: 각 채널을 독립적인 시퀀스로 취급

  ECG Lead II  ──→ [p1][p2][██][p4][p5][██][p7] ──→ Encoder ──→ MPM + Next-Pred
  EEG Fp1      ──→ [p1][██][p3][p4][██][p6][p7] ──→ Encoder ──→ MPM + Next-Pred
  Resp         ──→ [██][p2][p3][██][p5][p6][p7] ──→ Encoder ──→ MPM + Next-Pred

  ※ [██] = 마스킹된 패치    ※ 각 채널이 별도로 학습 (variate 간 상호작용 없음)
  ※ Loss = α·MPM + β·NextPred

═══════════════════════════════════════════════════════════════════
  Phase 2: Any-Variate (AV) Cross-Modal Training
═══════════════════════════════════════════════════════════════════

  입력: 같은 세션의 다변량을 하나의 시퀀스로 통합

  ┌─────────────────────────────────────────────────────────┐
  │  ECG   [p1][p2][██][p4]  ←── signal_type=0             │
  │  EEG   [██][██][██][██]  ←── variate-level masking      │
  │  Resp  [p1][██][p3][p4]  ←── signal_type=5             │
  └────────────────────┬────────────────────────────────────┘
                       │ Packed Batch
                       ▼
              ┌─────────────────┐
              │   Transformer    │
              │    Encoder       │
              │  (bidirectional) │
              └────────┬────────┘
                       │
          ┌────────────┼────────────┐
          ▼            ▼            ▼
     Reconstruction  Next-Pred  Cross-Modal
     (마스킹 복원)   (미래 예측)  (ECG→EEG 예측)

  ※ Variate-level masking: EEG 전체를 마스킹
    → ECG, Resp 정보로 EEG 복원 (Virtual Sensing)
  ※ Cross-Modal: 같은 시간대(time_id) 다른 modality 패치 간 예측
  ※ Contrastive: 같은 시간대 다른 modality의 표현을 의미 공간에서 정렬
  ※ Loss = α·MPM + β·(NextPred + γ·CrossRecon) + δ·Contrastive
```

#### Phase 1: Channel-Independent (CI) Pre-training

**목표**: 개별 채널의 시간적 패턴을 학습한다.

각 variate(채널)를 독립적인 시퀀스로 취급하여 **단일 신호 내의 시간적 구조**를 먼저 학습한다. 이 단계에서는 variate 간 상호작용을 고려하지 않으므로, 모델이 ECG의 심박 패턴, EEG의 수면 파형, 호흡의 주기성 등 **각 신호 고유의 표현**을 안정적으로 학습할 수 있을 것으로 기대한다.

- **Collate mode**: `ci` (channel-independent, 각 variate를 별도 sample로 처리)
- **학습 목표**: Masked Patch Modeling (α=1.0) + Next-Patch Prediction (β=1.0)
- **Cross-modal**: 비활성 (γ=0.0)

#### Phase 2: Any-Variate (AV) Cross-Modal Training

**목표**: 서로 다른 신호 간의 상관관계를 학습한다.

Phase 1의 checkpoint를 로드한 뒤, 다변량 세션 데이터에서 **cross-modal 학습**을 수행한다. 같은 환자의 동시 기록된 ECG, EEG, 호흡 등을 하나의 시퀀스로 묶어 처리하며, 한 신호의 정보로 다른 신호를 예측하는 과제를 통해 **신호 간 관계**를 학습하는 것을 목표로 한다.

- **Collate mode**: `any_variate` (같은 세션의 다변량을 하나의 packed sequence로)
- **학습 목표**: Masked Patch Modeling (α=0.7) + Next-Patch Prediction (β=0.3) + Cross-Modal Reconstruction (γ=1.0) + Cross-Modal Contrastive (δ=0.1)
- **Variate-level masking**: 전체 variate를 마스킹하여 Virtual Sensing 능력을 학습할 수 있도록 구성

### 3.2 Self-Supervised Learning 목표 함수

네 가지 자기지도 학습 과제를 복합 손실 함수로 통합한다:

$$\mathcal{L} = \alpha \cdot \mathcal{L}_{MPM} + \beta \cdot (\mathcal{L}_{Next} + \gamma \cdot \mathcal{L}_{CrossRecon}) + \delta \cdot \mathcal{L}_{Contrastive}$$

각 loss가 Packed Batch의 패치 시퀀스에서 어떻게 동작하는지 아래에 시각적으로 설명한다. 하나의 세션에서 ECG 1ch + EEG 2ch가 같은 row에 패킹된 예시를 기준으로 한다.

```
패치 인덱스:    [  0  ][  1  ][  2  ] [  3  ][  4  ][  5  ] [  6  ][  7  ][  8  ]
               ├─── ECG ch1 ────────┤ ├─── EEG ch1 ────────┤ ├─── EEG ch2 ────────┤
sample_id:     [  1  ][  1  ][  1  ] [  1  ][  1  ][  1  ] [  1  ][  1  ][  1  ]
variate_id:    [  1  ][  1  ][  1  ] [  2  ][  2  ][  2  ] [  3  ][  3  ][  3  ]
time_id:       [  0  ][  1  ][  2  ] [  0  ][  1  ][  2  ] [  0  ][  1  ][  2  ]
                 ↑                     ↑                     ↑
              같은 물리적 시간 t=0    같은 물리적 시간 t=0    같은 물리적 시간 t=0
```

#### (1) Masked Patch Modeling (MPM)

입력 패치 시퀀스의 일부를 랜덤 마스킹하고, **양방향(bidirectional) attention**으로 마스킹된 패치를 복원한다. BERT/MAE의 생체신호 적용에 해당한다.

```
원본:          [  0  ][  1  ][  2  ] [  3  ][  4  ][  5  ] [  6  ][  7  ][  8  ]
pred_mask:     [     ][ ████][     ] [ ████][     ][     ] [     ][ ████][     ]
                       mask                mask                    mask

Encoder 입력:  [  0  ][ MASK][  2  ] [ MASK][  4  ][  5  ] [  6  ][ MASK][  8  ]
                       ↕                ↕                          ↕
Encoder 출력 → head → [rec_1]         [rec_3]                    [rec_7]
                       ↕                ↕                          ↕
Loss:          MSE(rec_1, patch_1) + MSE(rec_3, patch_3) + MSE(rec_7, patch_7)
               ─────────────────────────────────────────────────────────────────
                              마스킹된 위치만 loss 계산
```

- **Patch-level masking**: mask_ratio=15%로 랜덤 패치 선택
- **Variate-level masking** (Phase 2): variate_mask_prob=30%로 전체 variate 마스킹 → Virtual Sensing 시뮬레이션

#### (2) Next-Patch Prediction

미래 패치를 예측하는 **인과적(causal) 과제**. 랜덤 horizon(1~max_horizon)으로 예측 거리를 다양화하여 단기~중기 시간적 의존성을 학습할 수 있도록 설계하였다.

```
variate_id=1 (ECG):    [  0  ][  1  ][  2  ]
                         │       │
                         │       └─── horizon=1 ──→ predict [  2  ]
                         └─── horizon=2 ──────────→ predict [  2  ]

variate_id=2 (EEG ch1): [  3  ][  4  ][  5  ]
                           │       │
                           │       └─── horizon=1 ──→ predict [  5  ]
                           └─── horizon=1 ──→ predict [  4  ]

Causal Attention (패치 i는 i 이전만 볼 수 있음):
   0 → [볼수있음]
   1 → [  0  ][볼수있음]
   2 → [  0  ][  1  ][볼수있음]
   ※ 다른 variate는 못 봄 (same sample_id 내에서만 attend)

Loss: MSE(next_head(encoded[i]), original_patch[i + H])
      same variate 조건: variate_id[i] == variate_id[i+H]
```

- Causal attention mask 적용으로 미래 정보 차단
- Same-variate 내에서 다음 패치 예측

#### (3) Cross-Modal Reconstruction (Phase 2)

같은 시간대(time_id)의 **다른 modality 패치**를 **raw 값으로 복원**하는 과제. 예를 들어, ECG 패치 정보로 동시간대 EEG 패치의 실제 파형을 예측하는 방식이다.

```
time_id:    0           1           2

ECG (v=1):  [patch_0] ─[patch_1] ─[patch_2]
               │           │           │
               │ cross     │ cross     │ cross
               │ predict   │ predict   │ predict
               ↓           ↓           ↓
EEG1(v=2):  [patch_3] ─[patch_4] ─[patch_5]
               │           │           │
               │ cross     │ cross     │ cross
               │ predict   │ predict   │ predict
               ↓           ↓           ↓
EEG2(v=3):  [patch_6] ─[patch_7] ─[patch_8]

매칭 조건: same (sample_id, time_id) AND different variate_id

예시 pairs:
  cross_head(encoded[0]) → predict patch_3 (ECG t=0 → EEG1 t=0)
  cross_head(encoded[0]) → predict patch_6 (ECG t=0 → EEG2 t=0)
  cross_head(encoded[3]) → predict patch_0 (EEG1 t=0 → ECG t=0)
  ...

Loss: MSE(cross_pred[i], original_patch[j])
```

- 신호 간 물리적 상관관계 학습을 기대 (ECG → PPG: 심박 → 맥파)
- Virtual Sensing의 기반이 될 수 있음: 누락된 신호를 다른 신호로부터 복원

#### (4) Cross-Modal Contrastive (Phase 2)

같은 시간대의 다른 modality **표현(representation)을 의미 공간에서 정렬**하는 과제. Cross-Modal Reconstruction이 raw 값 복원을 목표로 하는 것과 달리, Contrastive는 **"같은 시간의 ECG와 EEG는 같은 생리적 상태를 반영한다"**는 의미적 유사성을 학습한다.

```
time_id:    0           1           2

ECG (v=1):  [z_0]     ─[z_1]     ─[z_2]
               ↕ pull      ↕ pull      ↕ pull    ← Positive pairs
EEG1(v=2):  [z_3]     ─[z_4]     ─[z_5]          (같은 time_id, 다른 variate)
               ↕ pull      ↕ pull      ↕ pull
EEG2(v=3):  [z_6]     ─[z_7]     ─[z_8]

Positive (가깝게):               Negative (멀게):
  (z_0, z_3) same t=0, diff v     (z_0, z_4) diff t
  (z_0, z_6) same t=0, diff v     (z_0, z_1) same v
  (z_3, z_6) same t=0, diff v     (z_0, z_5) diff t
  (z_1, z_4) same t=1, diff v     ...나머지 모든 쌍
  ...

각 패치가 anchor가 되어 InfoNCE loss를 계산:
                                         exp(sim(z_0, z_3)) + exp(sim(z_0, z_6))
  anchor z_0 기준: Loss = -log ──────────────────────────────────────────────────
                                           Σ_{j≠0} exp(sim(z_0, z_j))
```

- SimCLR-style projection head (Linear→GELU→Linear)를 통해 encoder 출력을 contrastive 공간으로 투영
- Learnable temperature (log-parameterized, CLIP 방식)
- (3)과 같은 pair를 사용하지만, (3)은 **raw patch 복원**, (4)는 **표현 공간 정렬**이라는 점이 다름
- Downstream task 전이 시, 의미적으로 정렬된 표현이 fine-tuning 효율을 높일 수 있을 것으로 기대

#### 전체 Loss 흐름

```
Packed Batch Row
├── Encoder (bidirectional) ──→ encoded (B, N, d_model)
│   ├── head(encoded)              → reconstructed  ──→ (1) MPM Loss (α)
│   ├── cross_head(encoded)        → cross_pred     ──→ (3) CrossRecon Loss (γ)
│   └── contrastive_proj(encoded)  → z              ──→ (4) Contrastive Loss (δ)
│
└── Encoder (causal) ──→ encoded_causal
    └── next_head(encoded_h)       → next_pred      ──→ (2) NextPred Loss (β)

Total = α · (1) + β · ((2) + γ · (3)) + δ · (4)
```

| Loss | Attention 방향 | 매칭 조건 | 학습 목표 |
|------|---------------|----------|----------|
| (1) MPM | 양방향 | random mask | 시간적 패턴 복원 |
| (2) NextPred | causal | same variate, i→i+H | 미래 예측 |
| (3) CrossRecon | 양방향 | same time, diff variate (MSE) | Virtual Sensing |
| (4) Contrastive | 양방향 | same time, diff variate (InfoNCE) | 의미적 정렬 |

## 7. 기술적 의의

1. **Multi-modal 통합 사전학습**: 기존 연구(BIOT, LaBraM 등)가 단일 또는 제한된 모달리티에 집중한 반면, 다종의 생체신호를 **하나의 모델에서 통합 학습**하며 신호 종류 수에 제한 없이 확장 가능한 구조를 제안
2. **Cross-Modal Self-Supervised Learning**: 같은 시간대 다른 모달리티 예측(Reconstruction)과 표현 정렬(Contrastive)을 병행하여 **신호 간 관계 학습** — Reconstruction은 Virtual Sensing, Contrastive는 downstream 전이 품질 향상에 각각 기여할 것으로 기대
3. **2-Phase Curriculum**: Channel-Independent → Any-Variate 점진적 학습으로 **안정적이고 효과적인 multi-modal 표현 학습**을 도모
4. **효율적 가변 처리**: Packed Batch + bin-packing으로 **패딩 없는 가변 길이/채널 처리**가 가능하며, GPU 메모리 활용 향상을 기대
5. **임상 데이터 기반**: SNUH의 대규모 임상 데이터를 활용하여 **실제 의료 환경에서의 검증**을 계획

## 8. 논의 사항

### 8.1 입력 신호 길이

기존 생체신호 foundation model들은 대부분 10~30초의 짧은 입력을 사용하고 있다 (BIOT 10초, FoME 90초). 그러나 심방세동, 발작, 무호흡 등 **장기 패턴이 중요한 임상 과제**에서는 수 분 이상의 문맥이 필요할 수 있다. 현재 30초로 설정된 입력 길이를 최대 5분까지 확장하는 방안을 검토 중이며, **Variable-Length Training**(10초~5분 랜덤 윈도우)을 통해 다양한 시간 스케일에서 동작하는 모델을 지향하고 있다. 다만 입력 길이 증가는 패치 수 증가(self-attention O(N²))로 이어지므로, 모델 스케일업과 함께 진행될 필요가 있다.

### 8.2 생체신호 노이즈 처리

생체신호는 전극 탈락, 근전도 혼입, 움직임 아티팩트, baseline wandering 등 노이즈가 불가피하다. 모든 노이즈를 전처리로 제거하는 것은 현실적으로 어려우며, 과도한 전처리는 유의미한 신호까지 손실시킬 위험이 있다. 본 프로젝트는 다음과 같은 전략을 고려하고 있다:

- **최소 전처리**: 명백한 아티팩트(60Hz 전원 노이즈, 신호 saturation)만 제거
- **모델의 자체 강건성**: MPM 기반 self-supervised learning이 노이즈에 내재적으로 강건할 수 있음 (노이즈는 패치 간 일관성이 없어 모델이 자연스럽게 무시할 가능성)
- **Data Augmentation**: Gaussian Noise, Scale Jitter, Baseline Wander 등을 학습 시 의도적으로 추가하여 노이즈 내성을 높이는 방안
- **Robust Normalization**: PackedAbsMeanScaler가 outlier에 덜 민감한 정규화를 제공

### 8.3 모델 스케일링

Foundation model의 성능은 데이터 규모와 모델 크기에 크게 의존하는 것으로 알려져 있다. 현재 기본 설정(d_model=64, num_layers=2)은 프로토타이핑 및 검증용이며, 실제 사전학습 시에는 스케일업이 필요할 것으로 보인다.

| 규모 | 모델 설정 | 파라미터 수 | 적정 입력 길이 |
|------|----------|-----------|--------------|
| Small | d_model=256, L=6 | ~5M | 30초~1분 |
| Medium | d_model=512, L=12 | ~50M | 1분~3분 |
| Large | d_model=1024, L=24 | ~300M+ | 3분~5분 |

작은 모델로 아키텍처를 검증한 뒤 점진적으로 스케일업하는 방향을 고려하고 있다.

### 8.4 Downstream Task 선정

Foundation model의 가치를 입증하기 위해서는 다양한 downstream task에서의 성능을 보여줄 필요가 있다. 어떤 task를 선정할지는 데이터 가용성, 임상적 의의, 기존 벤치마크 존재 여부를 종합적으로 고려하여 결정해야 하며, 현재 검토 중인 후보는 다음과 같다:

| 후보 Task | 필요 신호 | 임상적 의의 | 비고 |
|-----------|----------|------------|------|
| 수술 중 저혈압 예측 | ABP, ECG, PPG | 마취과 의사결정 지원, 수술 중 합병증 예방 | VitalLab 기존 연구와 연계 가능 |
| 수술 후 섬망 예측 | EEG | 수술 후 합병증 조기 경고 | NPJ DM 게재 선례 있음 |
| 심정지 조기 예측 | ECG, ABP, PPG, Resp | ICU 환자 안전, 사망률 감소 | 24시간 예측, 높은 임상적 impact |
| 부정맥 분류 | ECG | 심장 질환 선별 | 공개 벤치마크(PTB-XL) 비교 가능 |
| Virtual Sensing | Cross-modal | 누락 채널 복원, 센서 장애 대응 | 본 모델 고유 기능, 차별화 포인트 |

선정 시 고려할 점:
- **VitalLab 데이터와의 정합성**: SNUH에서 확보 가능한 신호 종류와 라벨에 따라 실현 가능한 task가 달라질 수 있음
- **단기 vs 장기 예측**: 단기 task(부정맥 분류)는 현재 30초 입력으로 가능하나, 장기 task(심정지 예측)는 입력 길이 확장 또는 hierarchical 구조가 필요할 수 있음
- **Multi-modal 활용도**: 단일 신호로 해결 가능한 task보다, 여러 신호를 함께 써야 성능이 올라가는 task가 본 모델의 강점을 부각하는 데 유리할 것으로 보임
- **기존 SOTA 비교 가능성**: 공개 벤치마크가 있는 task를 최소 1~2개 포함하는 것이 바람직함

### 8.5 Cross-Modal Reconstruction의 생리학적 타당성

현재 Cross-Modal Loss(`NextPredictionLoss._cross_modal_loss`)는 같은 `(sample_id, time_id)`에서 서로 다른 `variate_id`를 가진 패치 쌍을 매칭하여 MSE reconstruction을 수행한다. 이때 **signal_type 구분 없이** 모든 cross-variate 쌍을 대상으로 하므로, ECG 패치로 EEG 파형을 복원하는 등 **생리학적으로 불가능한 reconstruction**이 학습 목표에 포함될 수 있다.

#### 생리학적 메커니즘 그룹 (Physiological Mechanism Group)

생체신호는 생성 메커니즘에 따라 그룹으로 분류할 수 있으며, **같은 그룹 내에서는 파형 간 인과적/물리적 관계가 존재**하여 cross-modal reconstruction이 타당하다.

| 그룹 | signal_type | 공유 메커니즘 | Cross-Modal 근거 |
|------|-------------|--------------|------------------|
| **Cardiovascular** | ECG (0), ABP (1), PPG (3) | 심장 전기활동 → 혈역학적 압력파 → 말초 혈관 용적 변화 | ECG R-peak → ABP systolic peak (pulse transit time), ECG→ABP 변환 연구(cuffless BP) 활발, PPG morphology와 ABP waveform 상관 |
| **Neural** | EEG (2) | 대뇌 피질 신경 활동 | EEG 채널 간 volume conduction으로 인접 전극 간 reconstruction 가능 |
| **Muscular** | EMG (4) | 골격근 전기 활동 | 같은 근육군 내 채널 간 |
| **Respiratory** | Resp (5) | 호흡 역학 (기류, 흉복부 운동) | Oronasal↔Thorax↔Abdomen 간 호흡 주기 공유 |

**핵심**: ECG↔ABP↔PPG는 모두 심혈관계의 산물이므로 cross-modal reconstruction이 생리학적으로 뒷받침된다. 반면 ECG↔EEG는 신호 생성 메커니즘, 진폭 스케일(mV vs μV), 주파수 대역이 근본적으로 달라 파형 수준의 reconstruction은 무의미하다.

#### Loss별 Cross-Modal 범위 차등 적용 (제안)

| Loss 유형 | Cross-Modal 범위 | 근거 |
|-----------|-----------------|------|
| **Reconstruction (MSE)**: `_cross_modal_loss` | **같은 mechanism group 내로 제한** | 파형 복원은 물리적 인과관계가 있어야 의미 있는 gradient 생성 |
| **Contrastive (InfoNCE)**: `CrossModalContrastiveLoss` | **전체 cross signal_type 허용** | Representation alignment이므로 "같은 시점의 생체 상태 공유" 수준의 고수준 연관성 학습 가능 (e.g., autonomic arousal → 심박↑ + 뇌파 각성 패턴) |

#### 구현 방안 (검토 필요)

`spatial_map.py`에 mechanism group 매핑 추가:

```python
MECHANISM_GROUP: dict[int, int] = {
    0: 0,  # ECG → Cardiovascular
    1: 0,  # ABP → Cardiovascular
    3: 0,  # PPG → Cardiovascular
    2: 1,  # EEG → Neural
    4: 2,  # EMG → Muscular
    5: 3,  # Resp → Respiratory
}
```

`_cross_modal_loss`에서 cross pair 매칭 시 `signal_type`의 mechanism group이 동일한 경우만 허용하도록 필터링 조건 추가.

#### 추가 고려 사항

- Heart-Brain Interaction (HEP: Heartbeat-Evoked Potential)이 존재하나, 이는 EEG에서 심박 타이밍에 lock된 미약한 전위로 full waveform reconstruction을 정당화하기엔 불충분
- Resp↔Cardiovascular 간에도 RSA(Respiratory Sinus Arrhythmia) 등 생리적 coupling이 있으나, reconstruction보다는 contrastive alignment가 적합
- 향후 mechanism group 간 경계를 soft하게 처리하는 방법도 검토 가능 (e.g., group 간 cross-modal weight를 낮추되 0으로 하지 않는 방식)

### 8.6 Few-Shot 적용을 위한 표현 품질

Foundation model이 few-shot으로 downstream task에 적용되려면, 사전학습된 표현 자체가 충분히 풍부해야 한다. 이를 위해 (1) 모델이 충분한 길이의 입력을 직접 처리하여 별도의 downstream aggregator 없이 linear probing만으로 예측이 가능한 구조를 지향하고, (2) Variable-Length Training으로 다양한 시간 스케일의 표현을 학습하며, (3) Cross-modal prediction으로 신호 간 관계까지 포착한 표현을 목표로 하고 있다.

또한 few-shot learning의 성능은 **support set의 품질**에 크게 좌우될 수 있다. 생체신호의 경우 같은 질환이라도 환자마다 파형이 다양하게 나타나므로, support set 구축 시 **임상의가 직접 전형적인(prototypical) 증상 신호를 선별**하는 과정이 필수적일 것으로 판단된다. 예를 들어, 심방세동 분류를 위한 support set에는 임상의가 "이것이 전형적인 AF 파형이다"라고 판단한 명확한 사례가 포함되어야 하며, 경계 사례(borderline)나 노이즈가 심한 구간은 배제하는 것이 바람직하다. 이는 모델의 few-shot 성능을 좌우하는 핵심 요소이며, **AI 엔지니어와 임상의의 협업**이 요구되는 지점이다.
